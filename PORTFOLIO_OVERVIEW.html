01 · Purpose

What this site is

A professional engineering portfolio for Phiwe Mhlope — Frontend & Full-Stack Engineer. Designed to communicate engineering maturity, production experience, and technical depth to recruiters and hiring managers. The aesthetic is inspired by modern developer documentation platforms (Vercel, Linear, Stripe Docs) rather than a generic personal portfolio.

05 · Features

Key UX & engineering features

  • Command Palette (Ctrl+K / Cmd+K) — keyboard-driven search across sections, both project case studies, resume, GitHub, and LinkedIn. Arrow-key navigation, Enter to select, Esc to close.
  • Project case study pages — each project opens at /projects/:slug with a project banner, sticky sidebar TOC, and 7 anchored sections: Overview, Problem, Architecture, Key Features, Challenges, Technical Highlights, Future Improvements.
  • Sidebar TOC with active tracking — Intersection Observer on the detail page highlights the current section in the sidebar as the user scrolls.
  • Active section detection in navbar — Intersection Observer with a tuned rootMargin highlights the correct nav item during scroll without false triggers.
  • Animated page transitions — Framer Motion AnimatePresence wraps the route outlet for fade transitions between home and project pages.
  • Responsive mobile drawer — spring-animated side drawer on mobile, with backdrop blur overlay and staggered menu item entrance.
  • Deployment-ready SPA routing — public/_redirects for Netlify and vercel.json rewrites for Vercel both configured.
  • Local resume — PDF served from public/, no external dependency or link rot risk.
  • SEO meta tags — Open Graph and Twitter card tags in index.html for social sharing previews.
  • Lazy image loading — project card and banner images use native loading="lazy".
  • Scroll-to-top — floating button appears at 400px scroll depth.

06 · Projects

Showcased projects

Afriverse — Full-Stack Digital Archive

A community-driven archive that aggregates African literary data from across the web into one structured, searchable platform. Author bios from Wikipedia, book covers from OpenLibrary and Google Books, interview thumbnails from YouTube — all pulled live. Features a full community contribution and admin moderation system, author profile claiming and verification, multi-dimensional discovery across six filter dimensions, literary awards tracking, and branded transactional email via Resend SMTP.

Architecture: Vite + React SPA with a custom in-memory pub/sub data cache (key-based invalidation). Supabase backend: PostgreSQL with row-level security, Supabase Auth, PostgREST API. A service layer normalises snake_case DB rows to camelCase app types. Four external APIs integrated client-side. Deployed on Vercel with CI/CD from GitHub.

ReactTypeScriptVite SupabasePostgreSQLReact Router v7 Tailwind CSSWikipedia APIOpenLibrary API Google Books APIResend SMTP

Product Store — MERN Inventory App

Full-stack MERN inventory management application. Express/Node REST API (4 endpoints, consistent HTTP status codes, error middleware) connected to MongoDB Atlas via Mongoose schema validation. React + TypeScript frontend. Express serves both the API and the Vite static build in production — single Render deployment, no CORS complexity.

Deployment targets

The site uses BrowserRouter (History API). The host must redirect all routes to index.html. Both configs are committed and ready.

HostConfig fileRule
Netlify public/_redirects /* /index.html 200
Vercel vercel.json rewrites: [{ source: "/(.*)", destination: "/index.html" }]

Previous deployment: phiwemhlopedev.netlify.app (Netlify). To redeploy to Netlify, connect the GitHub repo and set the build command to npm run build and publish directory to dist. No environment variables required.
